# Документация проекта

Здесь хранятся все материалы, относящиеся к процессу разработки: что обсуждалось, какие решения приняты, какие ТЗ выданы.

## Структура

- [`sessions/`](sessions/) — хронологические логи рабочих сессий и стримов. Каждая встреча или итерация vibe-coding с Claude — отдельный файл.
- [`specs/`](specs/) — самостоятельные технические задания и архитектурные документы (выделяются из сессий, когда становятся достаточно большими, чтобы жить отдельно).

## Соглашение по логированию

Каждая сессия работы над приложением документируется здесь файлом вида:

```
sessions/YYYY-MM-DD-краткое-имя.md
```

Файл содержит:
- **Контекст:** что было до сессии, какая задача стояла.
- **Что обсудили / сделали:** ключевые решения и изменения.
- **ТЗ на следующий шаг** (если применимо).
- **Вопросы и открытые пункты.**

Если несколько сессий за день — добавляйте порядковый номер: `2026-04-25-01-...`, `2026-04-25-02-...`.

## Как участвовать

- **Комментарии и предложения** — через Issues.
- **Правки** — через Pull Request.
- **Вопросы по ТЗ** — комментарием в Issue со ссылкой на конкретный документ.
