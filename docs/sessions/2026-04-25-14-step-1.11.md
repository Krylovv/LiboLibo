2. У прогресс-бара плеера должны быть поля слева и справа.

4. На экране подкастов размеры шрифтов должны совпадать с экраном Фида.

## Что произошло на самом деле

Я сделал PR #7, в котором попытался поправить заголовок через `lineLimit(3) + minimumScaleFactor(0.6)`. Заголовок продолжил вылезать. Я несколько раз перебирал решения, в том числе бегущую строку, но проблема возвращалась.

Параллельно внешний контрибьютор **@maxmaxme** прислал [PR #11](https://github.com/Krasilshchik3000/LiboLibo/pull/11) с **настоящей причиной**:

`BlurredBackdrop` лежал сиблингом в `ZStack { BlurredBackdrop; VStack { ... } }`. Внутри он использует `AsyncImage(...)` со `.scaledToFill()` — это значит, что картинка **может рендериться шире контейнера** (в этом смысл `scaledToFill`). Без `.clipped()` родительский `ZStack` растягивался под этот «излишек», и `VStack` с контентом получал ширину **больше ширины экрана**. Поэтому `padding(.horizontal, ...)` отнимал положенное от широченного контейнера, и greedy-элементы (GeometryReader-progress-bar, marquee-title) всё равно выходили за экран.

### Фикс из PR #11 (3 строки сути)

```swift
- ZStack {
-     BlurredBackdrop(url: ...)
-     VStack(spacing: 0) { ... }
- }
+ VStack(spacing: 0) { ... }
+ .frame(maxWidth: .infinity, maxHeight: .infinity)
+ .background {
+     BlurredBackdrop(url: ...)
+ }
```

И в `BlurredBackdrop`:
```swift
AsyncImage(url: url) { ... }
+   .clipped()
    .ignoresSafeArea()
```

`.background { }` подгоняет фон под размер foreground (VStack с явным `maxWidth: .infinity`, привязанный к safe area). `.clipped()` гарантирует, что `scaledToFill` не вылезет за границы.

PR #11 смержен в main. Мой PR #7 пересобран на новый main: я выбрал версию PlayerView от @maxmaxme и заменил неиспользуемый `VolumeRow + SystemVolumeSlider (MPVolumeView)` на свой кастомный `VolumeSlider` (см. ниже).

## Урок

Когда `padding` или `.frame` «не работают» в SwiftUI — почти всегда виноват предок, который захватывает больше места, чем должен. Прежде чем перебирать `.lineLimit`, `.fixedSize`, `.minimumScaleFactor` и кастомные маркизы, нужно проверить: **греется ли родитель, и не задирает ли его какой-то ребёнок-greedy** (`scaledToFill`, `GeometryReader` без явного frame, `ignoresSafeArea` в неправильном месте). Спасибо @maxmaxme за чистый минимальный фикс.
