## Что сделано

### Скрипт обогащения метаданных

[`scripts/refresh-podcast-metadata.py`](../../scripts/refresh-podcast-metadata.py) — Python-скрипт:
- Читает `docs/specs/podcasts-feeds.json`.
- Параллельно (12 потоков) скачивает каждый RSS, парсит `<channel><description>` (плюс fallback на `<itunes:summary>`) и максимальный `<pubDate>` среди `<item>`.
- Strip HTML тегов и сущностей.
- Записывает обратно в JSON две новые колонки на подкаст: `description`, `lastEpisodeDate` (ISO 8601).
- Дублирует результат в `LiboLibo/Resources/podcasts.json` (бандл).
- SSL отключён — RSS публичные, macOS Python иногда не может найти системный CA bundle.

Запускается вручную: `python3 scripts/refresh-podcast-metadata.py`. Перезапуск нужен раз в сутки/неделю или когда добавляются новые подкасты. Эта работа должна переехать на бэкенд (плановый шаг 2), скрипт — временный.

Все 44 фида обогатились без ошибок.

### Сегменты подкастов

[`Features/Podcasts/PodcastsView.swift`](../../LiboLibo/Features/Podcasts/PodcastsView.swift) — теперь `List` с тремя `Section`:

| Секция | Условие | Сортировка |
|---|---|---|
| **Выходят сейчас** | lastEpisodeDate ≥ now − 90 дней | по дате выпуска ↓ |
| **Недавно выходили** | lastEpisodeDate ∈ [now − 365д, now − 90д) | по дате ↓ |
| **Давно не выходят** | lastEpisodeDate < now − 365 дней (или не известна) | по дате ↓ |

Computation на in-memory массиве из `repository.podcasts` — мгновенно. Сетевых запросов нет (всё в JSON-бандле). `listStyle(.insetGrouped)`.

### Плеер: Apple-style прогресс-бар

[`Features/Player/PlayerView.swift`](../../LiboLibo/Features/Player/PlayerView.swift) → новый компонент `ProgressBar`:
- Тонкий трек **3pt** в idle (раньше было 4pt с большим thumb-ом).
- Без видимого thumb-а — простая капсула-track + капсула-fill.
- В момент скрабинга track «утолщается» до **6pt** (`isDragging` state) — Apple так делает на iOS 26.
- Скрабинг через `DragGesture(minimumDistance: 0)`, `seek(to:)` на end.
- Под слайдером — `0:00` слева и `−X:XX` справа (минус как у Apple).

### Регулятор громкости

Новый файл [`Features/Player/VolumeSlider.swift`](../../LiboLibo/Features/Player/VolumeSlider.swift):
```swift
struct SystemVolumeSlider: UIViewRepresentable {
    func makeUIView(context: Context) -> MPVolumeView {
        let view = MPVolumeView(frame: .zero)
        view.showsRouteButton = false
        view.tintColor = .white
        return view
    }
    func updateUIView(_ uiView: MPVolumeView, context: Context) {}
}
```

`MPVolumeView` — единственный санкционированный Apple способ менять системную громкость. В плеере я добавил `VolumeRow`: иконка `speaker.fill` слева, `SystemVolumeSlider` посередине, `speaker.wave.3.fill` справа (как в Apple Music / Now Playing widget).

> **Важно:** в iOS-симуляторе MPVolumeView не показывает реальный системный слайдер (симулятор не имеет аудиожелезки). На реальном устройстве это нативный системный слайдер с правильной анимацией thumb-а. Тестировать обязательно на iPhone.
