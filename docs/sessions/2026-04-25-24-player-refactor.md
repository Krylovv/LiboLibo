# 2026-04-25-24 — Рефактор PlayerView

## Контекст

После нескольких итераций над плеером (очередь, фид, лайаут) появилось несколько UX-правок: убрать отдельную кнопку описания, добавить меню с действиями, улучшить таймер сна. Параллельно откатили незрелую фичу голосовых заметок (её хотят сделать позднее).

## Что сделали

### PlayerService — расширение SleepTimer

Добавлены кейсы `.fortyfive` (45 мин) и `.untilEnd` (до конца выпуска). Добавлены два свойства:
- `pillLabel` — короткая метка для pill в плеере («Сон», «15м», «30м», «45м», «Эп.»).
- `menuLabel` — полная метка для меню («Выключено», «15 минут», …, «До конца выпуска»).

`playNextInContext()`: если `sleepTimer == .untilEnd`, вместо перехода к следующему выпуску плеер останавливается и сбрасывает таймер.

### PlayerView — рефактор

**Навигация внутри шита**: `PlayerView` теперь обёрнут в `NavigationStack(path: $path)`. Зарегистрированы `navigationDestination` для `Episode` и `Podcast` — это позволяет открывать экран выпуска или подкаста прямо внутри шита плеера с push-анимацией.

**Тапабельный заголовок**: тап по названию выпуска в `Titles` открывает `EpisodeNotesSheet` (описание) — вместо убранной кнопки `doc.text`.

**Меню `...` (`MoreMenu`)**: positioned справа над `UtilityRow`. Содержит:
- «Поделиться» — `ShareLink` с текстом «{podcastName} — {episodeTitle}».
- «Загрузить выпуск» / «Удалить загрузку» — через `DownloadService.toggle`.
- «Перейти к подкасту» — пушит `Podcast` в `path`.
- «Перейти к выпуску» — пушит `Episode` в `path`.

**Таймер сна (`SleepTimerMenu`)**: pill-кнопка заменена кастомным `Menu`-виджетом. Визуально выглядит как прежний pill (Capsule, тинт при активном состоянии). Список опций: Выключено / 15 / 30 / 45 минут / До конца выпуска. Активная опция отмечена галочкой.

**`UtilityRow`**: убрана кнопка `doc.text` (описание теперь по тапу на заголовок), убрана кнопка mic (голосовые заметки отложены). Остались: скорость, таймер сна (меню), кнопка загрузки, очередь.

## Открытые вопросы

- Голосовые заметки: отложены. Вернуться, когда определимся с форматом хранения (локально / в бэкенде).
- `PodcastDetailView` принимает `@Binding var path: NavigationPath` — при навигации из плеера используется локальный path плеера. Это означает, что история «назад» ведёт обратно в плеер, а не в основной стек — это ожидаемое поведение.
- «Перейти к подкасту»: если подкаст не найден в `repository.podcasts` (редкий кейс при холодном старте), пункт меню не показывается.
