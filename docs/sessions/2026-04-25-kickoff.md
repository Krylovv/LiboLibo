## Что мы поняли из стрима

### Контекст и команда

Делаем новое нативное мобильное приложение для подкастов «Либо-Либо». Существующее приложение в Telegram медленно работает за рубежом — главный болевой пункт. Команда:

- **Илья** — продукт и iOS UI.
- **Самат** — backend, админка, оплаты.
- **Егор** и другие — технические советники.

Подход — vibe-coding с Claude.

### Ключевые архитектурные решения

- **iOS — нативно на Swift/SwiftUI**, по Apple Human Interface Guidelines, с использованием liquid glass и стандартных анимаций. Кросс-платформа отвергнута: нет дизайнера, а гайды платформы «ведут по правильному пути». Android делаем потом — переносим через Claude.
- **Backend — Firebase + Firebase Functions** (serverless). Метаданные и файлы там же. Ключи держим только на сервере, не на клиенте.
- **Оплаты:** международные через Adaptive или RevenueCat, российские — CloudPayments.ru (отложено: нужна верификация). На MVP можно скипнуть.
- **Контент:** RSS-фид подкастов Либо-Либо из Apple Podcasts (издатель). Не все подкасты там есть — на старте этого ограничения достаточно.

### MVP — три экрана

1. **Фид** всех выпусков всех подкастов Либо-Либо.
2. **Подкасты** — список с возможностью подписаться.
3. **Профиль («Моё»)** — список подписок, недавние выпуски подписок, история прослушиваний.

### Плеер

Play/pause, ±10 сек, регулировка скорости, оффлайн-загрузка, история. Push-уведомления о новых выпусках и voice commentary — потом.

### Ключевая директива от Егора

Сначала «сырой работающий MVP» (RAW) — все компоненты связно работают, без красоты, без углубления в фичи. Потом полировка.

### Разделение работы

- **Илья** делает UI (этой сессией).
- **Самат** — backend + админка + оплаты параллельно.
- **Общий GitHub-репозиторий** — ✅ заведён (этот).

## ТЗ на первый шаг

### Цель

Поднять iOS-каркас приложения на SwiftUI: запускается в симуляторе, навигация работает, реальные подкасты Либо-Либо тянутся по RSS, плеер играет выпуск с базовыми контролами. Без бэкенда, без подписок-в-смысле-аккаунта, без оффлайна, без push, без voice commentary. Это «RAW», на который дальше навешиваются фичи.

### Стек

- Xcode-проект, iOS 18+, Swift, SwiftUI, AVFoundation (AVPlayer), URLSession, XMLParser/FeedKit для RSS.
- Архитектура — простая MVVM на `@Observable` / `@State`. Никаких лишних слоёв.

### Источник данных

RSS-ленты подкастов Либо-Либо (издатель в Apple Podcasts). На первом шаге — захардкоженный список из 2–3 RSS-URL, один универсальный парсер:

```
Episode { id, title, description, pubDate, audioURL, duration, podcastID }
Podcast { id, title, description, artworkURL, feedURL }
```

### Структура экранов (TabView с тремя вкладками)

1. **Фид** (`FeedView`)
   - Хронологический список выпусков из всех подкастов, сортировка по `pubDate desc`.
   - Ячейка: обложка подкаста, название выпуска, название подкаста, дата, длительность, кнопка play.
   - Тап по ячейке → `EpisodeDetailView` с описанием и большим плеером.

2. **Подкасты** (`PodcastsView`)
   - Сетка/список подкастов Либо-Либо.
   - Тап → `PodcastDetailView`: обложка, описание, кнопка «Подписаться» (на этом шаге — локально, в `UserDefaults`/`SwiftData`), список выпусков.

3. **Моё** (`ProfileView`)
   - Секция «Подписки» — подкасты, на которые подписан.
   - Секция «Недавние выпуски подписок».
   - Секция «История прослушиваний» (заполняется по факту запуска эпизода).
   - На первом шаге достаточно `SwiftData` для подписок и истории, без бэкенда.

### Плеер (`PlayerView` + `PlayerService`)

- Singleton-сервис на `AVPlayer` живёт всё время работы приложения.
- Mini-player фиксирован внизу над таб-баром, разворачивается в полноэкранный.
- Контролы: play/pause, ±10 сек, ползунок прогресса, скорость (0.8 / 1.0 / 1.2 / 1.5 / 2.0).
- Background audio + Now Playing Info Center + Remote Command Center (lock screen / control center).
- Capability: Background Modes → Audio.

### Дизайн

Системные компоненты SwiftUI (`NavigationStack`, `List`, `TabView`), SF Symbols для иконок, динамический шрифт (Dynamic Type), поддержка тёмной темы из коробки. Liquid glass — только там, где он берётся бесплатно из стандартных контролов (например, `.glassBackgroundEffect()` для mini-player на iOS 18+).

### Что НЕ делаем на этом шаге (важно — чтобы не утонуть)

- Backend, Firebase, API. Подкасты — напрямую с RSS.
- Оффлайн-скачивание (ставим на 2-й шаг).
- Push-уведомления.
- Аутентификацию и аккаунт.
- Платежи и подписку как монетизацию.
- Voice commentary.
- Android.
- App Store-сборку, signing-конфигурацию, certificates.

### Внешние зависимости и риски

- Нужен Apple Developer аккаунт — для запуска на устройстве. На симуляторе можно без него. На этом шаге достаточно симулятора.
- RSS-фиды Либо-Либо — нужно собрать список URL (можно вытащить из Apple Podcasts по издателю). На вход первой итерации — захардкоженный массив.

### Definition of Done для шага 1

- Приложение запускается в Xcode-симуляторе.
- На вкладке «Фид» видны реальные выпуски минимум двух подкастов Либо-Либо.
- Тап по выпуску запускает аудио, mini-player виден, контролы работают.
- На вкладке «Подкасты» видны подкасты, подписка локально сохраняется.
- На вкладке «Моё» отражаются подписки и история.
- Код в общем git-репозитории.
