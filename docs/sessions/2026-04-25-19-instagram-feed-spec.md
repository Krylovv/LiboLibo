# 2026-04-25 — Сессия 19. Instagram-лента: спека и план Фазы A

## Контекст

В приложении уже есть лента эпизодов («Фид»), экраны подкастов, плеер,
профиль, поиск. Бэкенд (api/) тянет RSS Transistor по cron. Возникла
идея сделать ещё один экран — лента из Instagram-аккаунта Либо-Либо
с галереями и рилсами.

## Что обсудили / решили

Через цикл уточняющих вопросов выяснили:

- **Продуктовая роль ленты:** социальная витрина студии — то, что Либо-Либо
  постит в IG, должно быть видно внутри приложения, чтобы пользователь
  не уходил в Instagram.
- **Курирование:** не зеркалим IG автоматически. Каждый пост проходит
  ручную модерацию через web-админку, к нему привязывается CTA одного из
  трёх типов: эпизод подкаста, подкаст или внешняя ссылка. Транслируется
  «почти всё» что есть в IG-фиде, но строго через подтверждение.
- **UX в iOS:** вертикальная лента карточек (как Twitter/Telegram), карусели
  через свайп, рилсы — инлайн-плеер. CTA-кнопка под caption.
- **Видео:** скачиваем mp4 к себе на Railway volume и отдаём со своего
  CDN. Истекающие `media_url` от Graph API не подходят для долгосрочной
  раздачи; проксирование на лету упрётся в лимиты Meta.
- **Источник:** Instagram Graph API через привязку IG к Facebook Page
  (доступ есть). Скрейпинг исключён.

## Артефакты

Записаны и закоммичены:

1. **Спека:** [docs/specs/step-03-instagram-feed.md](../specs/step-03-instagram-feed.md) — архитектура целиком (4 фазы):
   - A. Backend infra (Prisma, Graph API клиент, collector, cron).

   - C. Admin UI + публичный API.
   - D. iOS (новая вкладка «Лента», карточка, видеоплеер, навигация по CTA).
2. **План реализации Фазы A:** [docs/specs/step-03-instagram-feed-plan-phase-a.md](../specs/step-03-instagram-feed-plan-phase-a.md) — задачи 1–8 в TDD-стиле, по аналогии с существующим `src/transistor/`.

## ТЗ на следующий шаг

Фаза A. Идти по плану `step-03-instagram-feed-plan-phase-a.md`:

- Tasks 1–6 — чисто код, без внешних зависимостей. Можно делать без токена IG.
- Task 7 — локальный smoke; **нужен long-lived Page access token** из Meta
  Business Suite. Получить заранее.
- Task 8 — Railway: добавить переменные окружения и создать новый Cron
  Service с командой `npm run refresh:instagram` и расписанием `*/30 * * * *`.

После успешного выполнения Фазы A — отдельная сессия на план Фазы B
(media-downloader + ffmpeg + статик-раздача).
