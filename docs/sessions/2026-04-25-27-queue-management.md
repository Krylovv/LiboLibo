# 2026-04-25-27: Queue management

## Контекст

Очередь воспроизведения уже хранится в `PlayerService.feedContext`. Нужно было дать пользователю возможность управлять ею из интерфейса: добавлять выпуски через контекстное меню и удалять/переставлять их в экране очереди.

## Что сделали

### PlayerService

Добавили три метода управления очередью:

- `playNext(_ episode:)` — вставляет выпуск сразу после текущего (если уже есть в очереди — сначала убирает оттуда).
- `removeFromQueue(_ episode:)` — удаляет выпуск из `feedContext` (текущий удалить нельзя).
- `moveInQueue(fromOffsets:toOffset:)` — переставляет выпуски в секции «Далее» (индексы относительно неё, не абсолютные).

### EpisodeListItem (FeedView.swift)

Полностью переписан с inline-версткой (без `EpisodeRow`). В правом нижнем углу — меню «...» (`Menu`) с опциями:

- Загрузить / Удалить загрузку
- Поделиться
- Воспроизвести далее
- Перейти к выпуску
- Перейти к подкасту (только если передан `onNavigateToPodcast`)

Компонент принимает два новых опциональных колбэка: `onPlayNext` и `onNavigateToPodcast`.

### QueueSheetView

В секции «Далее»:

- Свайп влево → удаление (`.swipeActions(edge: .trailing)` с `role: .destructive`).
- Перетаскивание за иконку «≡» справа → изменение порядка. Реализовано кастомным `DragGesture` с захардкоженной высотой ряда `rowHeight = 52`: считаем `delta = translation.height / rowHeight`, двигаем элемент в локальной копии (`localAfterItems`), на `.onEnded` сохраняем порядок через `player.setQueueAfter(...)`. Native `.onMove` не используется, чтобы handle был всегда виден без перехода в edit mode.

### Остальные экраны

`PodcastDetailView`, `ProfileView`, `SearchView` — во все вызовы `EpisodeListItem` добавлены `onPlayNext` и (где применимо) `onNavigateToPodcast`.

## Технические детали

- `Menu` внутри `Button`-обёртки строки работает в `List` — Menu перехватывает свои тапы до outer Button.
- `Array.move(fromOffsets:toOffset:)` из стандартной библиотеки Swift для реорганизации секции «Далее».
- Индексы в `moveInQueue` — относительные (offset внутри `after`-слайса), конвертируются обратно перед сохранением в `feedContext`.
