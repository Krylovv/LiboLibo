curl -s 'https://itunes.apple.com/search?term=либо-либо&entity=podcast&limit=50&country=ru' \

## Вопрос Ильи: что бы я изменил в ТЗ для не-программистов?

> Если бы это приложение вайб-кодили люди, которые ничего не понимают в программировании, ты бы поменял что-нибудь в этом ТЗ?

**Короткий ответ:** да, существенно. Текущий ТЗ написан для инженера, который читает между строк. Для не-программистов нужна параллельная «Дорожка 2».

**Длинный ответ — что конкретно меняется:**

### 1. Убрать жаргон реализации
Сейчас в ТЗ — SwiftUI, AVFoundation, MVVM, `@Observable`, Background Modes, FeedKit, singleton-сервис. Не-программист не пишет код руками — он направляет Claude. Эти слова ему не нужны и пугают. Заменить на наблюдаемое поведение.

### 2. Acceptance criteria через поведение, не через структуру кода
Вместо «Singleton-сервис на AVPlayer» — «когда тапаешь по выпуску, аудио играет; свернёшь приложение — продолжает играть». То же ограничение, но проверяется глазами и ушами, а не code review.

### 3. Меньшие шаги
Каждый шаг — самостоятельно запускаемый, визуально проверяемый, ~10–15 минут на итерацию. Вместо одного «iOS-каркас MVP» разложить:
- 1.0 — пустое приложение запускается с тремя пустыми вкладками;
- 1.1 — на «Фиде» список из 5 фейковых выпусков (моки);
- 1.2 — список настоящие выпусков одного подкаста;
- 1.3 — тап играет звук;
- 1.4 — контролы плеера (play/pause, ±10);
- ...

Не-программисту критичны частые маленькие победы — иначе теряет ориентацию.

### 4. Recovery-инструкции
«Если Claude сломал сборку — вот команда rollback. Если фича не работает — вот шаблон, как описать баг Claude». Программисты получают это бесплатно из инстинктов; не-программисты тонут на первом сломанном билде.

### 5. On-ramp по тулингу
«Как установить Xcode, как открыть симулятор, как запустить Claude Code, как давать промпты». Без этого первая остановка = последняя.

### 6. Готовые промпты-стартеры
Маленькая библиотека: «Промпт для шага 1.0 — скопируй Claude». Снижает барьер входа на порядок.

### 7. Меньше выбора
Текущий ТЗ предлагает «UserDefaults или SwiftData» — для не-программиста это выбор без контекста. Лучше выбрать одно и при необходимости переиграть позже.

### Вывод

Делать ли Дорожку 2 сейчас? **Не сейчас.** Сначала пройти шаг 1, посмотреть, где конкретно тонут не-программисты в зрительском чате стрима, и потом написать Дорожку 2 как ремикс — на реальных болях, не в вакууме.

Илье как продакт-руководителю инженерная версия удобнее: он направляет Claude и должен ловить, когда что-то идёт не туда. Параллельная Дорожка 2 — для зрителей стрима, кто захочет повторить путь.
