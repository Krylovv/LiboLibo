## Что сделано

### Плеер

- [`Services/PlayerService.swift`](../../LiboLibo/Services/PlayerService.swift) — `@MainActor @Observable` обёртка над `AVPlayer`:
  - State: `currentEpisode`, `isPlaying`, `currentTime`, `duration`, `rate` (0.8 / 1.0 / 1.25 / 1.5 / 2.0).
  - API: `play(_:)`, `togglePlayPause()`, `skip(by:)`, `seek(to:)`.
  - Audio session — `.playback` (играет в беззвучном режиме, продолжает в фоне).
  - Periodic time observer обновляет `currentTime`/`duration` 2 раза в секунду.
  - **Lock screen / Control Center**: подключены `MPRemoteCommandCenter` (play, pause, toggle, skip ±10 s, scrub) и `MPNowPlayingInfoCenter` (название, подкаст, прогресс, скорость).

### UI

- [`Features/Player/MiniPlayerView.swift`](../../LiboLibo/Features/Player/MiniPlayerView.swift) — мини-плеер: обложка 44×44, два текстовых ряда, кнопки −10 s и play/pause, фон `regularMaterial` со скруглением 14.
- [`Features/Player/PlayerView.swift`](../../LiboLibo/Features/Player/PlayerView.swift) — полноэкранный плеер: большая обложка, заголовок, прогресс-слайдер с временами, кнопки ±10 s + play/pause, меню скоростей.
- [`App/RootView.swift`](../../LiboLibo/App/RootView.swift) — `safeAreaInset(edge: .bottom)` с мини-плеером, появляется при `currentEpisode != nil`. Тап → `sheet(isPresented:)` с полноэкранным плеером. Плавная анимация появления.
- [`Features/Feed/FeedView.swift`](../../LiboLibo/Features/Feed/FeedView.swift) и [`Features/Podcasts/PodcastDetailView.swift`](../../LiboLibo/Features/Podcasts/PodcastDetailView.swift) — каждая ячейка обёрнута в `Button { player.play(episode) }` со стилем `.plain`.

### Конфиг

- В pbxproj добавлен `INFOPLIST_KEY_UIBackgroundModes = audio;` для обоих конфигураций — даёт capability «Background Modes → Audio».
- `App/LiboLiboApp.swift` создаёт `PlayerService` в `@State` и пробрасывает через `.environment(_:)`.

## Архитектурные решения

- **Один экземпляр на жизнь приложения**, через `@Environment` доступен любому View.
- **Все мутации состояния — на главной нити** (`@MainActor`). Колбэки `MPRemoteCommandCenter` и `addPeriodicTimeObserver` диспатчатся в Main через `Task { @MainActor in ... }` и `MainActor.assumeIsolated`.
- **Никаких сторонних зависимостей** — `AVFoundation` + `MediaPlayer` из коробки.
- **Без `deinit`-cleanup**: сервис живёт всё время приложения (через `@State` в `@main App`), поэтому очистка observer-а не критична. Исключённый ради компиляции с актором.
