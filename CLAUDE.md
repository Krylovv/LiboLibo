# CLAUDE.md

Этот файл — инструкции для Claude Code и любых других AI-ассистентов, работающих в этом репозитории. Они автоматически подхватываются при запуске Claude Code в корне проекта. Люди-контрибьюторы тоже должны следовать этим правилам.

## Что это за репозиторий

Нативное мобильное приложение «Либо-Либо» (iOS + бэкенд). Разрабатывается публично в режиме vibe-coding. Подробности — в [README.md](README.md).

## Главное правило: документируй каждую сессию

Каждая работа над репо (стрим, парная сессия с Claude, самостоятельный заход) **обязательно** заканчивается двумя артефактами:

1. **Лог сессии** в `docs/sessions/YYYY-MM-DD-NN-краткое-имя.md`. Содержит:
   - Контекст: что было до сессии, какая стояла задача.
   - Что обсудили / решили / сделали.
   - ТЗ или план на следующий шаг (если применимо).
   - Открытые вопросы.

2. **Спека**, если результат сессии — самостоятельный технический документ (план фазы, API-контракт, ADR). Кладётся в `docs/specs/`.

Если несколько сессий в один день — порядковый номер: `2026-04-25-01-...`, `2026-04-25-02-...` и т.д.

Имена сессий — короткие, в kebab-case, на латинице или транслите: `step-2-backend-kickoff`, не `Шаг-2-стартовая-сессия`.

## Язык документации

Русский. Сохраняем оригинальные английские технические термины (имена полей, фреймворков, статусы) и опционально добавляем перевод в скобках, если смысл неочевиден.

Исключение: `api/` (исходники бэкенда) — комментарии и идентификаторы на английском, как обычно в коде.

## Структура репозитория

```
LiboLibo/                  # iOS-приложение (Swift, SwiftUI)
LiboLibo.xcodeproj/        # Xcode-проект
api/                       # Бэкенд (Node.js, Express, Prisma)
docs/                      # Вся документация
  sessions/                # Логи рабочих сессий
  specs/                   # ТЗ, архитектурные документы, OpenAPI
  screenshots/             # Скриншоты приложения
README.md
SECURITY.md
CLAUDE.md                  # ← вы здесь
```

## Безопасность

См. [SECURITY.md](SECURITY.md). Коротко:

- Никаких секретов в коммитах: `.env`, ключи, токены, сертификаты.
- iOS — секреты в Keychain.
- Бэкенд — секреты только в Railway Variables.
- Repo публичный, всё попадает в датасеты — действуйте соответствующе.

## Технологический стек

- **iOS:** Swift, SwiftUI, AVFoundation, SwiftData, URLSession, XMLParser. iOS 18+.
- **Бэкенд:** Node.js, TypeScript, Express, Prisma, Postgres (Railway plugin), Railway Cron Service, fast-xml-parser, vitest. Деплой — Railway.
- **Локальная разработка бэкенда:** Docker Compose (Postgres + API).
- **Источник контента:** RSS-фиды Либо-Либо на Transistor.fm.

## Что делать в начале новой сессии Claude Code

1. Прочитай свежий лог сессии в `docs/sessions/` — последний по дате.
2. Прочитай актуальную спеку фазы в `docs/specs/`.
3. Уточни у пользователя цель сессии.
4. В конце — обязательно создай лог сессии (см. выше).

## Ритуал завершения задачи (iOS)

После каждой задачи, которая трогает iOS-код, **обязательно**:

1. Собрать: `xcodebuild -project LiboLibo.xcodeproj -scheme LiboLibo -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17' -configuration Debug build` — ожидать `** BUILD SUCCEEDED **`.
2. Установить и запустить в booted-симуляторе: `xcrun simctl install booted <path-to-LiboLibo.app>` + `xcrun simctl launch booted test.libolibo.ru`.
3. Закоммитить изменения (включая лог сессии) и сделать `git push`.

Сама просьба сделать что-то и есть просьба собрать и запушить — отдельного подтверждения не требуется. Если задача трогает только `api/` или только документацию — iOS-сборку пропускаем, но коммит и пуш делаем. Если сборка падает — не коммитим, чиним и пересобираем.

## Чего не делать

- Не добавляй emoji в код и документацию, если пользователь явно не попросил.
- Не правь iOS и бэкенд в одной сессии без явной нужды — это разные слои, разные ответственные, разные ритмы релизов.
- Не создавай README.md внутри подпапок без необходимости — структура и так документирована здесь и в `docs/README.md`. Исключение — `api/README.md` (инструкции по запуску бэкенда локально).
