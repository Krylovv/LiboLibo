# Безопасность

Этот репозиторий публичный. Всё, что в него попадает, — видно всем и сразу попадает в зеркала, поисковики и обучающие датасеты. Поэтому строгие правила.

## Что НЕ кладём в репозиторий

Никаких:

- API-ключей, токенов, паролей, OAuth-секретов;
- приватных сертификатов и провиженов: `*.p12`, `*.mobileprovision`, `*.keystore`, `*.pem`, `*.key`;
- конфигов с секретами: `.env`, `GoogleService-Info.plist`, `firebase-adminsdk-*.json`, `*-credentials.json`;
- SSH-ключей, AWS / GCP / Anthropic / OpenAI токенов;
- персональных данных пользователей и внутренних логов.

Эти паттерны прописаны в [`.gitignore`](.gitignore). Если файл с секретом не попадает под них — добавь паттерн.

## Как храним секреты

- **iOS-клиент.** Чувствительные значения — в Keychain, не в `UserDefaults` и не в коде. Если нужен публичный ключ для third-party SDK — он публичен, можно держать в коде, но всё равно лучше через `.xcconfig` без коммита значений.
- **Node.js-бэкенд (Самат).** Секреты — в переменных окружения у платформы развёртывания. В репо — только `.env.example` со списком имён переменных и комментариями, **без значений**.
- **Сетевые вызовы** — только по HTTPS.
- **Логи** — никаких токенов, тел авторизационных запросов, JWT, куков.

## На стримах

- Не показываем содержимое `.env` и любых файлов с ключами на экране.
- Перед открытием Claude Code / терминала проверяем, что в активной директории и в окне терминала нет ключей.
- Если ключ всё-таки промелькнул — считаем его скомпрометированным и **сразу ротируем**. Удаление из истории git вторично.

## Если ключ всё-таки утёк

1. **Ротируйте немедленно.** Выпустите новый, отзовите старый. Не «подумаем потом» — прямо сейчас.
2. Сообщите в команду (Илье или Самату).
3. Удалите из истории git (`git filter-repo` или `git rebase`). Это не отменяет компрометации, только убирает из публичного дерева.

## Автоматические проверки

- На репозитории включено **GitHub Secret Scanning** (бесплатно для публичных репо). При попытке запушить распознанный секрет — пуш блокируется.
- Перед коммитом полезно прогонять локально:
  ```bash
  git diff --staged | grep -iE 'token|secret|key|password|api[_-]?key'
  ```
- Рекомендуем установить [`gitleaks`](https://github.com/gitleaks/gitleaks):
  ```bash
  brew install gitleaks
  gitleaks protect --staged
  ```
